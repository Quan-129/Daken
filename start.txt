## SCHEMA 09: DATABASE MODEL (Kiến Trúc Dữ Liệu - Supabase/PostgreSQL)
Để tối ưu hóa truy vấn và đồng bộ Realtime, hệ thống phân tách thành các bảng sau:

### 1. Table: `profiles` (Thông tin người dùng)
- `id`: uuid (Primary Key - Link with Auth.users)
- `display_name`: varchar(50)
- `avatar_url`: text
- `neural_id`: varchar(15) (Unique - ID định danh kiểu Cyberpunk)
- `current_level`: int (1-5 tương ứng N1-N5)
- `created_at`: timestamp

### 2. Table: `user_stats` (Chỉ số hiệu năng - Tách biệt để update nhanh)
- `profile_id`: uuid (Foreign Key -> profiles.id)
- `total_sp`: bigint (Total Score Points)
- `avg_wpm`: float4
- `max_wpm`: int4
- `avg_accuracy`: float4
- `total_words_mastered`: int4
- `last_active`: timestamp

### 3. Table: `vocabulary` (Kho từ vựng JLPT)
- `id`: serial (Primary Key)
- `level`: int2 (1-5)
- `kanji`: varchar(100)
- `kana`: varchar(100)
- `meaning_vi`: text
- `romaji`: varchar(100) (Dùng để check typing logic)
- `tags`: text[] (e.g: ["verb", "noun", "common"])

### 4. Table: `match_history` (Lịch sử trận đấu - Big Data)
- `id`: uuid (Primary Key)
- `profile_id`: uuid (Foreign Key)
- `mode`: varchar(20) (Study/Online/Custom)
- `level`: int2
- `wave_reached`: int2
- `score`: int4
- `wpm`: float4
- `accuracy`: float4
- `played_at`: timestamp

### 5. Table: `user_settings` (Cấu hình cá nhân - Đồng bộ đa thiết bị)
- `profile_id`: uuid (Foreign Key)
- `audio_cfg`: jsonb {bgm: float, sfx: float, vocal: float, tts_rate: float}
- `theme_cfg`: jsonb {neon_color: string, font_size: int}
- `keybinds`: jsonb (Custom phím tắt nếu có)

## SCHEMA 10: IN-MEMORY GAME STATE (Mô Hình Trạng Thái Trận Đấu)
Để đảm bảo tốc độ phản hồi 0ms (Zero Latency), logic game sẽ quản lý data cục bộ trước khi sync:

### 1. `EngineState` (Trạng thái lõi)
- `input_buffer`: string (Chuỗi người dùng đang gõ)
- `current_target_id`: int (ID của từ đang cần gõ)
- `wave_start_time`: timestamp
- `combo_multiplier`: float (x1.0 -> x5.0)

### 2. `CombatEntity` (Dữ liệu thực thể địch - Vocabulary trên màn hình)
- `uuid`: string
- `vocab_data`: Object (Ref to Vocabulary Table)
- `position`: {x, y, velocity}
- `status`: "active" | "destroyed" | "escaping"
- `health`: int (Số ký tự còn lại cần gõ)

### 3. `AnalyticsStore` (Tính toán chỉ số realtime)
- `raw_keystrokes`: int (Tổng số lần nhấn phím)
- `error_count`: int
- `perfect_words`: int (Số từ gõ không sai lỗi nào)
- `timeline`: Array<{timestamp, type: "hit" | "miss"}> (Dùng để vẽ biểu đồ hiệu suất sau trận)

## SCHEMA 12: OPTIMIZATION & PERFORMANCE (Tối Ưu Hóa Trải Nghiệm)
Để đạt chuẩn "Scientific & Optimized", hệ thống cần áp dụng các kỹ thuật sau:

### 1. Asset Memory Management (Quản lý Bộ nhớ)
- **Object Pooling:** Tái sử dụng các thực thể (Enemy/Card) trong Wave thay vì tạo mới liên tục để tránh Garbage Collection (GC) gây giật lag.
- **Lazy Loading:** Chỉ load bộ từ vựng của Level đang chọn (N1-N5), không load toàn bộ Database lên Client.

### 2. Network Latency Mitigation (Giảm độ trễ mạng)
- **Client-side Prediction:** Trong mode Online, client sẽ dự đoán kết quả gõ và hiển thị ngay lập tức, sau đó mới đối chiếu với Server.
- **Debouncing Input:** Tối ưu hóa việc gửi dữ liệu phân tích (Analytics) về server theo cụm (batch) thay vì gửi từng nhịp gõ phím.

### 3. Rendering Performance (Hiệu suất hiển thị)
- **Offscreen Canvas:** Thực hiện các phép tính toán đồ họa phức tạp ở background thread.
- **RequestAnimationFrame:** Đảm bảo toàn bộ hiệu ứng visual chạy ở mức 60 FPS ổn định.
- **CSS Hardware Acceleration:** Sử dụng `transform: translate3d` và `will-change` cho các hiệu ứng neon di chuyển nhanh.

### 4. Code Quality & Scalability (Chất lượng Code)
- **Modular Architecture:** Chia nhỏ logic thành các Feature độc lập (theo cấu trúc `src/features/`).
- **Unit Testing:** Viết test cho `TypingEngine` để đảm bảo logic tính xác suất và WPM luôn chuẩn xác 100%.
